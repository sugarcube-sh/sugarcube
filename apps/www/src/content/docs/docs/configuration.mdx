After running `init`, sugarcube works with zero configuration via sensible defaults. This page covers how to customize things when the defaults don't fit.

<SiteAside type="tip">
    Most of these options can also be passed as CLI flags if you don't want to create a config file — see the [CLI reference](/docs/reference/cli-commands) for details. For a complete reference of every config option, see the [configuration schema reference](/docs/reference/configuration-schema).
</SiteAside>

## Creating a config file

Create a `sugarcube.config.{ts|js}` in your project root:

```ts title="sugarcube.config.ts"
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({});
```

## Configuring utility classes

Sugarcube can generate utility classes from your design tokens. Utilities are configured in the `utilities.classes` key of your config file. See [CSS Utilities](/docs/utilities) for full details.

## Changing where CSS files are written

<SiteAside type="note">
    `variables.path` and `utilities.path` only apply when using the CLI. The Vite plugin serves token variables and utilities as virtual modules and doesn't write them to disk.
</SiteAside>

Set `variables.path` and `utilities.path` to specify where CSS files are written:

```ts title="sugarcube.config.ts" ins={5-8}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  variables: {
    path: "src/styles/tokens.css"
  },
  utilities: {
    path: "src/styles/utilities.css"
  }
});
```

## Changing where components are copied to

By default, components are copied to `src/components/ui` (or `components/ui` if there's no `src/` directory).

```ts title="sugarcube.config.ts" ins={4}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  components: "components"
});
```

## Changing where CUBE CSS is copied to

By default, CUBE CSS files are written to your styles directory (e.g. `src/styles`). To put them somewhere else:

```ts title="sugarcube.config.ts" ins={4}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  cube: "src/css/cube"
});
```

## Configuring fluid spacing and typography

To adjust the 'slope' of the scaling curve for fluid calculations, configure the viewport range:

```ts title="sugarcube.config.ts" ins={5-10}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  variables: {
    transforms: {
      fluid: {
        min: 375,
        max: 1440
      }
    }
  }
});
```

For more on fluid spacing and typography in sugarcube, see [fluid spacing and typography](/docs/fluid-space-and-type).


## CSS Cascade Layers

When using the CLI, you can wrap generated CSS in `@layer` blocks:

```ts title="sugarcube.config.ts" ins={5,8}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  variables: {
    layer: "tokens"
  },
  utilities: {
    layer: "utilities"
  }
});
```

For full details on cascade layers, see [CSS Cascade Layers](/docs/cascade-layers).


## Polyfilling modern CSS color formats

To polyfill modern CSS color formats for older browsers, set `colorFallbackStrategy` to `"polyfill"`:

```ts title="sugarcube.config.ts" ins={5-7}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  variables: {
    transforms: {
      colorFallbackStrategy: "polyfill"
    }
  }
});
```

For more on color formats in sugarcube, see [color formats](/docs/color-formats).


## Configuring permutations

Permutations control how modifier contexts map to CSS output. By default, sugarcube auto-generates permutations from your resolver's modifiers:

- Default contexts → `:root`
- Non-default contexts → `[data-{modifierName}="{context}"]`

For custom selectors or media queries, define explicit permutations:

```ts title="sugarcube.config.ts" ins={5-10}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  variables: {
    permutations: [
      { input: { theme: "light" }, selector: ":root" },
      { input: { theme: "dark" }, selector: "[data-theme=\"dark\"]" },
    ]
  }
});
```

Each permutation has:

- **`input`**: Object mapping modifier names to context values (missing modifiers use defaults)
- **`selector`**: CSS selector for this permutation
- **`atRule`**: Optional at-rule wrapper (e.g., `@media (prefers-color-scheme: dark)`)
- **`path`**: Optional output file path (defaults to `variables.path`)

### Using media queries

To use `prefers-color-scheme` or other media queries:

```ts title="sugarcube.config.ts"
variables: {
  permutations: [
    { input: { theme: "light" }, selector: ":root" },
    {
      input: { theme: "dark" },
      selector: ":root",
      atRule: "@media (prefers-color-scheme: dark)"
    },
  ]
}
```

For full theming documentation, see [theming](/docs/theming).
