After running `init`, sugarcube works with zero configuration via sensible defaults. This page covers how to customize things when the defaults don't fit.

<SiteAside type="tip">
    Most of these options can also be passed as CLI flags if you don't want to create a config file — see the [CLI reference](/docs/reference/cli-commands) for details. For a complete reference of every config option, see the [configuration schema reference](/docs/reference/configuration-schema).
</SiteAside>

## Creating a config file

Create a `sugarcube.config.{ts|js}` in your project root:

```ts title="sugarcube.config.ts"
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({});
```

## Configuring utility classes

Sugarcube can generate utility classes from your design tokens. Utilities are configured in the `utilities` key of your config file. See [CSS Utilities](/docs/utilities) for full details.

## Changing where CSS files are written

<SiteAside type="note">
    `cssRoot`, `variables`, and `utilities` only apply when using the CLI. The Vite plugin serves token variables and utilities as virtual modules and doesn't write them to disk.
</SiteAside>

Set `output.cssRoot` to change the base output directory (defaults to `src/styles` or `styles`):

```ts title="sugarcube.config.ts" ins={5}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  output: {
    cssRoot: "public/css"
  }
});
```

Token variables, utilities, and CUBE CSS are all output relative to this directory. For example, with the default `cssRoot` of `src/styles`:

- `variables` → `src/styles/global`
- `utilities` → `src/styles/utilities`
- `cube` → `src/styles`

You can override individual paths if needed:

```ts title="sugarcube.config.ts" ins={6-7}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  output: {
    cssRoot: "src/styles",
    variables: "src/tokens",
    utilities: "src/css/utilities"
  }
});
```

## Changing generated filenames

By default, sugarcube generates `tokens.variables.gen.css` and `utilities.gen.css`. To change these:

```ts title="sugarcube.config.ts" ins={5-6}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  output: {
    variablesFilename: "tokens.css",
    utilitiesFilename: "utils.css"
  }
});
```

## Changing where components are copied to

By default, components are copied to `src/components/ui` (or `components/ui` if there's no `src/` directory).

```ts title="sugarcube.config.ts" ins={5}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  output: {
    components: "components"
  }
});
```

## Changing where CUBE CSS is copied to

By default, CUBE CSS files are written to `cssRoot` (e.g. `src/styles`). To put them somewhere else:

```ts title="sugarcube.config.ts" ins={5}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  output: {
    cube: "src/css/cube"
  }
});
```

## Configuring fluid spacing and typography

To adjust the 'slope' of the scaling curve for fluid calculations, configure the viewport range:

```ts title="sugarcube.config.ts" ins={5-8}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  transforms: {
    fluid: {
      min: 375,
      max: 1440
    }
  }
});
```

For more on fluid spacing and typography in sugarcube, see [fluid spacing and typography](/docs/fluid-space-and-type).


## CSS Cascade Layers

When using the CLI, you can wrap generated CSS in `@layer` blocks:

```ts title="sugarcube.config.ts" ins={5-8}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  output: {
    layers: {
      variables: "base",
      utilities: "utilities"
    }
  }
});
```

For full details on cascade layers, see [CSS Cascade Layers](/docs/cascade-layers).


## Polyfilling modern CSS color formats

To polyfill modern CSS color formats for older browsers, set `colorFallbackStrategy` to `"polyfill"`:

```ts title="sugarcube.config.ts" ins={5}
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  transforms: {
    colorFallbackStrategy: "polyfill"
  }
});
```

For more on color formats in sugarcube, see [color formats](/docs/color-formats).
