Fluid spacing and typography create responsive designs that adapt naturally to different screen sizes. Sugarcube supports fluid dimensions using the standard DTCG `dimension` type with an extension.

## Adding fluid dimension tokens

Use a standard `dimension` token with the `sh.sugarcube.fluid` extension. The `$value` serves as a fallback for tools that don't understand the extension, while the extension defines the fluid scaling range:

```json title="src/design-tokens/spacing.json"
{
  "spacing": {
    "$type": "dimension",
    "sm": {
      "$value": { "value": 16, "unit": "px" },
      "$extensions": {
        "sh.sugarcube": {
          "fluid": {
            "min": { "value": 0.5, "unit": "rem" },
            "max": { "value": 1, "unit": "rem" }
          }
        }
      }
    }
  }
}
```

This approach keeps your tokens DTCG-compliant. Other tools will see a valid `dimension` token and use the `$value` as a static fallback, while Sugarcube generates fluid `clamp()` values from the extension.

<SiteAside type="note">
    The namespace `sh.sugarcube` follows the [DTCG $extensions specification](https://www.designtokens.org/tr/2025.10/format/#extensions) using reverse domain notation. The `fluid` key is nested within this namespace.
</SiteAside>

## Legacy format (deprecated)

<SiteAside type="caution">
    The `fluidDimension` type is deprecated and will be removed in v2.0. Please migrate to the extension-based format above.
</SiteAside>

The older `fluidDimension` type is still supported but not recommended:

```json title="src/design-tokens/spacing.json"
{
  "spacing": {
    "$type": "fluidDimension",
    "sm": {
      "$value": {
        "min": { "value": 0.5, "unit": "rem" },
        "max": { "value": 1, "unit": "rem" }
      }
    }
  }
}
```

This format is not part of the DTCG specification and won't be recognized by other design token tools.

## Adjusting the scaling rate

`fluidDimension` tokens generate CSS variables using `clamp()` functions:

```css
--spacing-sm: clamp(0.5rem, 0.5rem + 0.57vw, 1rem);
```

The preferred value scales smoothly between min and max across the viewport range. By default, sugarcube uses 320px (min) and 1200px (max). The viewport range determines the scaling slope:

- **Wider range** (e.g., 320px to 1440px) = gentler slope, slower scaling
- **Narrower range** (e.g., 375px to 1200px) = steeper slope, faster scaling

Customize viewport boundaries in your config:

```ts title="sugarcube.config.ts"
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  variables: {
    transforms: {
      fluid: {
        min: 375,
        max: 1440
      }
    }
  }
});
```
