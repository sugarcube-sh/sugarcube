# CLI Commands

## init

Set up a new sugarcube project with design tokens and CSS architecture.

```bash
npx @sugarcube-sh/cli init
```

**What it does:**
- Creates project structure
- Installs design token starter kits
- Sets up CSS architecture
- Configures build tools
- Creates config file if you customize settings

## generate

Generate CSS variables and utility classes from your design tokens.

```bash
sugarcube generate [options]
```

**What it does:**
- Processes design token files
- Generates CSS variables
- Creates utility classes
- Outputs CSS files

**Works without a config file** - auto-discovers resolver files and uses sensible defaults. All settings can be overridden via CLI flags.

## validate

Validate design token files for W3C compliance.

```bash
sugarcube validate
```

**What it does:**
- Checks token syntax
- Validates W3C format compliance
- Reports errors and warnings
- Ensures token consistency

**Works without a config file** - auto-discovers resolver if no paths specified.

## components

Install copy-to-use components for your framework.

```bash
sugarcube components
```

**What it does:**
- Lists available components
- Installs selected components
- Copies component files
- Updates project structure

**Works without a config file** - uses default component directory or accepts `--components-dir` flag.

## cube

Add CUBE CSS architecture to your project.

```bash
sugarcube cube
```

**What it does:**
- Installs CUBE CSS compositions
- Adds utility classes
- Sets up CSS architecture
- Configures build integration

**Works without a config file** - uses default CSS directory or accepts `--cube-dir` flag.
