## Commands

- **`init`** - Set up a new project
- **`generate`** - Generate CSS variables and utility classes from your design tokens. Supports `--watch` for continuous regeneration during development. Use this if you are *not* using the Vite plugin.
- **`validate`** - Validate design token files for W3C compliance
- **`components`** - Install copy-to-use components for your framework
- **`cube`** - Add CUBE CSS architecture to your project

## Typical Workflow

1. **Initialize**: `npx @sugarcube-sh/cli init` - Sets up design tokens, optionally adds CUBE CSS and components, installs dependencies
2. **Generate** CSS (if not using the Vite plugin): `sugarcube generate` - Processes tokens, outputs CSS
3. **Add components** as needed: `sugarcube components`
4. **Validate** tokens: `sugarcube validate`
