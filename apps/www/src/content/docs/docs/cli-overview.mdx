## Commands

- **`init`** - Set up a new project
- **`generate`** - Generate CSS variables and utility classes from your design tokens. Use this if you are *not* using the [Vite plugin](/docs/vite-plugin).
- **`validate`** - Validate design token files for W3C compliance
- **`components`** - Install copy-to-use components for your framework
- **`cube`** - Add CUBE CSS architecture to your project

## Typical Workflow

1. **Initialize**: `npx @sugarcube-sh/cli init` - Sets up starter kit, detects framework, installs dependencies
2. **Generate** CSS (if not using the Vite plugin): `npx @sugarcube-sh/cli generate` - Processes tokens, outputs CSS
3. **Add components** as needed: `npx @sugarcube-sh/cli components`
4. **Validate** tokens: `npx @sugarcube-sh/cli validate`
