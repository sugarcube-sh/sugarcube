Complete reference for all configuration options in `sugarcube.config.ts` (or `sugarcube.config.js` for JavaScript projects).

<SiteAside type="note">
    A config file is **optional**. Sugarcube auto-discovers a resolver and uses default paths when no config is present. Create a config file when you need to customize output paths, transforms, or utilities.
</SiteAside>

## Type safety

Use the `defineConfig` helper for full autocomplete and type checking. Import from the package you already have installed:

- **If you use the Vite plugin**: import from `@sugarcube-sh/vite`
- **If you use the CLI only**: import from `@sugarcube-sh/cli`

```ts
// When using the Vite plugin
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  // ... your config (with full autocomplete!)
});
```

```ts
// When using the CLI only
import { defineConfig } from "@sugarcube-sh/cli";

export default defineConfig({
  // ... your config
});
```

This works for both TypeScript and JavaScript files — no JSDoc annotations needed.

## resolver

**Type:** `string`
**Required:** No (auto-discovered if omitted)

Path to your resolver document — a JSON file that defines your token sources and theming structure.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  resolver: "src/design-tokens/tokens.resolver.json"
});
```

If omitted, sugarcube will recursively search your project for `*.resolver.json` files. Only specify this if you have multiple resolver files and need to choose a specific one.

For details on resolver documents, see [Resolver Document](/docs/resolver).

## variables

**Type:** `object`
**Optional**

Configuration for CSS variables output.

### `variables.path`

**Type:** `string`
**Default:** `"src/styles/variables.gen.css"` or `"styles/variables.gen.css"`

Output path for the CSS variables file. Default adapts based on whether your project has a `src/` directory.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  variables: {
    path: "dist/tokens.css"
  }
});
```

### `variables.layer`

**Type:** `string`
**Optional**

CSS cascade layer name. When set, output is wrapped in an `@layer` block.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  variables: {
    layer: "tokens"
  }
});
```

<SiteAside type="note">
    This option only applies when using the CLI. For Vite plugin users, configure layers via `unoOptions.outputToCssLayers` instead. See [CSS Cascade Layers](/docs/cascade-layers).
</SiteAside>

### `variables.transforms`

**Type:** `object`
**Optional**

Token transformation options.

#### `variables.transforms.fluid`

**Type:** `{ min: number, max: number }`
**Default:** `{ min: 320, max: 1200 }`

Viewport range for fluid typography calculations. Tokens with `$type: "fluidDimension"` will scale between these viewport widths.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  variables: {
    transforms: {
      fluid: {
        min: 375,
        max: 1440
      }
    }
  }
});
```

#### `variables.transforms.colorFallbackStrategy`

**Type:** `"native" | "polyfill"`
**Default:** `"native"`

How to handle colors that can't be represented in sRGB.

- **`"native"`**: Use CSS color functions directly (modern browsers only)
- **`"polyfill"`**: Generate fallback values for older browsers

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  variables: {
    transforms: {
      colorFallbackStrategy: "polyfill"
    }
  }
});
```

### `variables.permutations`

**Type:** `Permutation[]`
**Optional**

Defines how modifier contexts map to CSS selectors. Each permutation selects a resolver input and specifies where to output the result.

When not defined, sugarcube auto-generates permutations:
- Default contexts → `:root`
- Non-default contexts → `[data-{modifierName}="{context}"]`

Each permutation has:
- **`input`**: Which modifier contexts to resolve (see below)
- **`selector`**: CSS selector (string or array of strings for comma-joined selectors)
- **`atRule`**: Optional at-rule wrapper (e.g., `@media (prefers-color-scheme: dark)`)
- **`path`**: Optional output file path (defaults to `variables.path`)

#### How `input` works

The `input` object maps modifier names to context values. You only need to specify the modifiers that differ from their defaults — any modifier not listed uses its `default` value from the resolver.

For example, given a resolver with `theme` (default: `"light"`) and `density` (default: `"normal"`):

```ts
// These two are equivalent — density defaults to "normal"
{ input: { theme: "dark", density: "normal" }, selector: "..." }
{ input: { theme: "dark" }, selector: "..." }
```

This means you typically only specify one modifier per permutation. The default for everything else is implicit:

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  variables: {
    permutations: [
      // All defaults (theme: light, density: normal)
      { input: { theme: "light" }, selector: ":root" },
      // Only theme changes — density stays at its default
      { input: { theme: "dark" }, selector: "[data-theme=\"dark\"]" },
      // Only density changes — theme stays at its default
      { input: { density: "compact" }, selector: "[data-density=\"compact\"]" },
    ]
  }
});
```

<SiteAside type="tip">
  When no permutations are defined, sugarcube auto-generates them from your resolver's modifiers — defaults go to `:root`, non-defaults get `[data-{modifier}="{context}"]` selectors. Define explicit permutations when you need custom selectors, media queries, or per-brand output files.
</SiteAside>

Permutations can target different output files:

```ts
permutations: [
  { input: { brand: "base" }, selector: ":root", path: "dist/base.css" },
  { input: { brand: "ocean" }, selector: ":root", path: "dist/ocean.css" },
]
```

Selectors can be arrays for compound selectors (comma-joined in output):

```ts
permutations: [
  {
    input: { theme: "dark" },
    selector: [
      "[data-color-mode=light][data-light-theme*=dark]",
      "[data-color-mode=dark][data-dark-theme*=dark]"
    ]
  },
]
```

See [Theming](/docs/theming) for full documentation on permutations.

## utilities

**Type:** `object`
**Optional**

Configuration for utility classes output.

### `utilities.path`

**Type:** `string`
**Default:** `"src/styles/utilities.gen.css"` or `"styles/utilities.gen.css"`

Output path for the utility classes CSS file. Default adapts based on whether your project has a `src/` directory.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  utilities: {
    path: "dist/utilities.css"
  }
});
```

### `utilities.layer`

**Type:** `string`
**Optional**

CSS cascade layer name. When set, output is wrapped in an `@layer` block.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  utilities: {
    layer: "utilities"
  }
});
```

### `utilities.classes`

**Type:** `Record<string, UtilityConfig | UtilityConfig[]>`
**Optional**

Maps CSS property names to utility class configurations.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  utilities: {
    classes: {
      color: {
        source: "color.*",
        prefix: "text"
      },
      padding: {
        source: "space.*",
        prefix: "p",
        directions: ["all", "x", "y"]
      }
    }
  }
});
```

#### Utility config options

<Table>
| Option | Type | Description |
|--------|------|-------------|
| `source` | `string` | **Required.** Glob pattern matching token paths (e.g., `"color.*"`) |
| `prefix` | `string` | Custom prefix for class names |
| `directions` | `string[]` | Directional variants: `"top"`, `"right"`, `"bottom"`, `"left"`, `"x"`, `"y"`, `"all"` |
| `stripDuplicates` | `boolean` | Remove duplicate values |
</Table>

#### Multiple configs per property

Provide an array for multiple configurations:

```ts
utilities: {
  classes: {
    color: [
      { source: "color.brand.*", prefix: "text" },
      { source: "color.semantic.*", prefix: "text" }
    ]
  }
}
```

## components

**Type:** `string`
**Default:** `"src/components/ui"` or `"components/ui"`

Directory where components are copied when running the `components` command.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  components: "src/ui"
});
```

## cube

**Type:** `string`
**Default:** `"src/styles"` or `"styles"`

Directory where CUBE CSS files are written when using the `cube` command.

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  cube: "src/css/cube"
});
```

## Default values

When options are omitted, sugarcube uses these defaults (paths adapt based on `src/` directory presence):

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  variables: {
    path: "src/styles/variables.gen.css",
    transforms: {
      fluid: { min: 320, max: 1200 },
      colorFallbackStrategy: "native"
    }
  },
  utilities: {
    path: "src/styles/utilities.gen.css"
  },
  components: "src/components/ui",
  cube: "src/styles"
});
```

## Examples

### Minimal config

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  resolver: "src/design-tokens/tokens.resolver.json"
});
```

### With custom paths

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  resolver: "src/design-tokens/tokens.resolver.json",
  variables: {
    path: "dist/tokens.css"
  },
  utilities: {
    path: "dist/utilities.css"
  },
  components: "src/components"
});
```

### With transforms

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  resolver: "src/design-tokens/tokens.resolver.json",
  variables: {
    transforms: {
      fluid: { min: 375, max: 1440 },
      colorFallbackStrategy: "polyfill"
    }
  }
});
```

### With utilities

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  resolver: "src/design-tokens/tokens.resolver.json",
  utilities: {
    classes: {
      color: {
        source: "color.*",
        prefix: "text"
      },
      "background-color": {
        source: "color.*",
        prefix: "bg"
      },
      padding: {
        source: "space.*",
        prefix: "p",
        directions: ["all", "x", "y"]
      }
    }
  }
});
```

### Complete example

```ts
import { defineConfig } from "@sugarcube-sh/vite";

export default defineConfig({
  resolver: "src/design-tokens/tokens.resolver.json",

  variables: {
    path: "src/styles/variables.gen.css",
    layer: "tokens",
    transforms: {
      fluid: { min: 375, max: 1440 },
      colorFallbackStrategy: "polyfill"
    },
    permutations: [
      { input: { theme: "light" }, selector: ":root" },
      { input: { theme: "dark" }, selector: "[data-theme=\"dark\"]" },
    ]
  },

  utilities: {
    path: "src/styles/utilities.gen.css",
    layer: "utilities",
    classes: {
      color: { source: "color.*", prefix: "text" },
      padding: { source: "space.*", prefix: "p", directions: ["all", "x", "y"] }
    }
  },

  components: "src/components/ui",
  cube: "src/styles"
});
```
